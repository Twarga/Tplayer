# Tplayer v0.3.0 — UI Polish, Consistency & Performance

## Overview
No new headline features. This release is a full-depth pass to make every screen
feel premium and consistent, eliminate performance hazards for large libraries,
and fix every visual inconsistency found in the v0.2.0 codebase.

---

## Release Version
**0.3.0** — minor release (backward-compatible improvements, no breaking changes)

---

## Issues Found in v0.2.0 Codebase

### UI Inconsistencies
| Location | Problem |
|---|---|
| `AlbumView` | Shows a generic `<Disc>` icon — no album art even when track covers exist |
| `ArtistView` | Shows a generic `<Users>` icon — no artist art |
| `PlaylistListView` | Shows a single first-letter initial, no cover mosaic |
| `PlaylistDetailView` | Same single-letter header, no cover mosaic |
| `MiniPlayerBar` | Cover art thumbnail has no `border-radius` — inconsistent with NowPlayingPanel |
| `MiniPlayerBar` | Volume uses a raw `<input type="range">` — not the shared Radix `<Slider>` |
| `NowPlayingPanel` | "Lyrics" tab clickable but renders nothing — dead UI |
| `NowPlayingPanel` | Panel collapse mixes `w-0`/`w-[336px]` class toggle with `panelMotion` x-animation — they fight |
| `LibraryView` | Loading skeleton uses `animate-pulse` not the design-system `.skeleton` shimmer |
| `HomeView` | "YouTube imports" section shows ALL tracks, not YouTube-sourced only |
| `HomeView` | `MoreVertical` button on import rows has no handler |
| `HomeView` | Greeting "Good evening" is hardcoded — not time-aware |
| `HomeView` | No "Favorites" section |
| `SettingsView` | Shows `Tplayer v0.1.0` — stale version string |
| `TopBar` | Bell notification icon has no handler — dead UI |
| `QueueView` | Currently-playing row has no distinct visual treatment |
| `Sidebar` | Inline `Create Playlist` modal is a raw `<div>` — not using Radix Dialog |
| `PlaylistListView` | Inline create dialog is a raw `<div>` — duplicates Sidebar's modal |
| All views | No `loading="lazy"` on any `<img>` — hurts initial render |
| `AlbumView` / `ArtistView` | No entrance stagger animations for grid items |

### Performance Issues
| Location | Problem |
|---|---|
| `AlbumView` | Full grid rendered, no virtualization — freezes with 200+ albums |
| `ArtistView` | Same — no virtualization |
| `PlaylistListView` | Same — no virtualization |
| `AlbumView` / `ArtistView` | `handlePlayAlbum` loops `api.queue.add(id)` — N sequential IPC calls |
| `HomeView` | `recentTracks`, `highlightTracks`, `importRows` slices computed fresh each render — no `useMemo` |
| `LibraryView` + `HomeView` | Both call `loadTracks()` on mount — double IPC call at startup |
| `QueueView` | `reorder` writes to DB immediately on every drag — should debounce |

### 🔲 Task 17 — Unified Radix Dialog for Create Playlist

#### Create `src/renderer/components/ui/create-playlist-dialog.tsx`
```tsx
import * as Dialog from '@radix-ui/react-dialog'

interface Props {
  open: boolean
  onClose: () => void
  onConfirm: (name: string) => Promise<void>
}

export function CreatePlaylistDialog({ open, onClose, onConfirm }: Props) {
  const [name, setName] = useState('')

  const handleSubmit = async () => {
    if (!name.trim()) return
    await onConfirm(name.trim())
    setName('')
    onClose()
  }

  return (
    <Dialog.Root open={open} onOpenChange={(v) => !v && onClose()}>
      <Dialog.Portal>
        <Dialog.Overlay className="fixed inset-0 z-50 bg-black/50 animate-fade-in" />
        <Dialog.Content className="fixed left-1/2 top-1/2 z-50 -translate-x-1/2 -translate-y-1/2 w-80 bg-surface-elevated border border-border-default rounded-xl p-6 shadow-modal animate-scale-in">
          <Dialog.Title className="text-lg font-semibold text-primary mb-4">
            Create Playlist
          </Dialog.Title>
          <input
            autoFocus
            value={name}
            onChange={(e) => setName(e.target.value)}
            onKeyDown={(e) => e.key === 'Enter' && handleSubmit()}
            placeholder="Playlist name"
            className="w-full h-10 px-3 rounded-md bg-input-bg border border-input-border text-primary placeholder:text-tertiary focus:border-accent focus:outline-none mb-4"
          />
          <div className="flex justify-end gap-2">
            <button onClick={onClose} className="px-4 py-2 rounded-md text-sm text-secondary hover:text-primary hover:bg-surface-2 transition-all active:scale-95">
              Cancel
            </button>
            <button onClick={handleSubmit} className="px-4 py-2 rounded-md text-sm bg-accent text-background hover:bg-accent-hover transition-all active:scale-95">
              Create
            </button>
          </div>
        </Dialog.Content>
      </Dialog.Portal>
    </Dialog.Root>
  )
}
```
